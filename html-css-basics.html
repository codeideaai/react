HTML 与 CSS

学习 React 前需要掌握的页面结构与布局基础

通过商品卡片示例回顾语义化 HTML、表单、盒模型、Flexbox、Grid 和响应式设计

React 负责根据数据生成页面，但最终显示在浏览器中的仍然是 HTML 和 CSS。组件拆分是否合理、表单是否容易使用、页面能否适配不同屏幕，都离不开这两项基础能力。

本章不会覆盖 HTML 和 CSS 的全部内容，而是集中讲解 React 页面开发中最常用的知识。

1 学习目标

完成本章后，你应该能够：

  • 使用语义化 HTML 组织页面结构
  • 正确使用链接、图片、按钮和表单元素
  • 理解标签、属性、嵌套和文档结构
  • 使用常见 CSS 选择器设置样式
  • 理解层叠、继承、优先级和盒模型
  • 使用 Flexbox 完成一维布局
  • 使用 Grid 完成二维布局
  • 使用定位处理悬浮元素
  • 使用媒体查询实现响应式页面
  • 说明 HTML/CSS 写法迁移到 JSX 时的主要差异

2 HTML 负责页面结构

HTML 的作用是描述内容的结构和含义，而不是决定页面最终长什么样。

一个基础页面通常包含下面这些部分：

<!doctype html>
<html lang="zh-CN">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>商品列表</title>
  </head>
  <body>
    <header>
      <h1>在线商城</h1>
    </header>

    <main>
      <section>
        <h2>热门商品</h2>
        <p>查看本周最受欢迎的商品。</p>
      </section>
    </main>

    <footer>
      <p>© 2026 在线商城</p>
    </footer>
  </body>
</html>

在 React 项目中，index.html 通常只保留一个根容器，组件会把页面内容渲染到这个容器中。但组件内部仍然需要使用合理的 HTML 标签。

3 语义化标签

语义化标签可以直接表达内容的作用。浏览器、搜索引擎、屏幕阅读器和开发者都能更容易理解页面结构。

3.1 页面区域

常用的页面结构标签包括：

标签 用途
header 页面或区域的头部
nav 主要导航链接
main 页面主要内容，一个页面通常只有一个
section 围绕一个主题组织的内容区域
article 可以独立存在的完整内容
aside 与主要内容相关的补充内容
footer 页面或区域的底部

例如，一张商品卡片可以使用 article：

<article class="product-card">
  <img src="keyboard.jpg" alt="黑色机械键盘" />
  <h2>机械键盘</h2>
  <p>支持有线和蓝牙连接。</p>
  <p>¥299</p>
  <button type="button">加入购物车</button>
</article>

3.2 标题层级

标题从 h1 到 h6 表示内容层级，而不是字体大小。

<h1>React 学习课程</h1>

<section>
  <h2>基础回顾</h2>

  <section>
    <h3>HTML 与 CSS</h3>
  </section>
</section>

不要为了得到较小的字体而跳过标题层级。字体大小应该交给 CSS 控制。

提示选择标签的方法

先思考“这段内容是什么”，再选择标签。如果只是为了布局且没有明确语义，可以使用 div；如果是一段行内文本，可以使用 span。

4 标签、属性与嵌套

大多数 HTML 元素由开始标签、内容和结束标签组成：

<p>这是一段商品说明。</p>

属性为元素提供额外信息：

<a href="/products" title="查看全部商品">商品列表</a>

元素必须正确嵌套：

<!-- 正确 -->
<p>学习 <strong>React</strong> 开发。</p>

<!-- 错误：标签交叉 -->
<p>学习 <strong>React</p></strong>

4.1 常见全局属性

  • id：页面中某个元素的唯一标识
  • class：为一个或多个元素设置样式分组
  • title：补充说明，不应作为主要信息的唯一来源
  • data-*：保存自定义数据
  • hidden：隐藏元素
<article
  id="product-1"
  class="product-card featured"
  data-product-id="1"
>
  商品内容
</article>

5 链接、图片与按钮

5.1 链接

链接用于导航到另一个地址：

<a href="/products/1">查看商品详情</a>

打开新窗口时，要同时考虑安全属性：

<a
  href="https://example.com"
  target="_blank"
  rel="noreferrer"
>
  访问外部网站
</a>

5.2 图片

alt 用于描述图片内容。当图片加载失败或用户使用屏幕阅读器时，它非常重要。

<img
  src="keyboard.jpg"
  alt="带有白色背光的黑色机械键盘"
  width="640"
  height="480"
/>

如果图片纯粹用于装饰，可以使用空的 alt：

<img src="decoration.svg" alt="" />

5.3 按钮

按钮用于执行操作：

<button type="button">加入购物车</button>

不要使用 div 模拟按钮。真正的 button 默认支持键盘操作、焦点和辅助技术。

重要链接与按钮的区别
  • 前往另一个页面或地址：使用 a
  • 提交、删除、打开弹窗等操作：使用 button

6 表单基础

表单用于收集用户输入。常见元素包括 input、select、textarea 和 button。

<form>
  <div>
    <label for="product-name">商品名称</label>
    <input
      id="product-name"
      name="productName"
      type="text"
      required
    />
  </div>

  <div>
    <label for="category">商品分类</label>
    <select id="category" name="category">
      <option value="keyboard">键盘</option>
      <option value="mouse">鼠标</option>
    </select>
  </div>

  <div>
    <label for="description">商品说明</label>
    <textarea id="description" name="description" rows="4"></textarea>
  </div>

  <button type="submit">保存商品</button>
</form>

6.1 label 与输入框

label 的 for 应与输入框的 id 对应。点击标签时，浏览器会自动聚焦对应的输入框。

不要只使用 placeholder 代替标签，因为用户输入后提示文字会消失。

6.2 常见输入类型

<input type="text" />
<input type="email" />
<input type="password" />
<input type="number" />
<input type="date" />
<input type="checkbox" />
<input type="radio" />

选择正确的 type 可以得到基础校验，并让移动设备显示更合适的键盘。

6.3 按钮类型

  • submit：提交表单
  • reset：重置表单
  • button：普通操作按钮

表单内没有声明 type 的按钮通常默认是提交按钮。不是用于提交时，应该明确写 type="button"。

7 CSS 负责页面表现

CSS 用来控制颜色、字体、间距、大小和布局。

一条 CSS 规则由选择器和声明块组成：

.product-card {
  color: #1f2937;
  background-color: #ffffff;
  border-radius: 12px;
}
  • .product-card 是选择器
  • color、background-color 是属性
  • #1f2937、#ffffff 是属性值

8 CSS 选择器

8.1 基础选择器

/* 标签选择器 */
button {
  cursor: pointer;
}

/* 类选择器 */
.product-card {
  padding: 16px;
}

/* ID 选择器 */
#featured-product {
  border-color: gold;
}

日常组件样式通常优先使用类选择器。ID 选择器优先级较高，不适合大量复用。

8.2 组合选择器

/* product-card 内部的 h2 */
.product-card h2 {
  font-size: 1.25rem;
}

/* 同时拥有两个类 */
.product-card.featured {
  border-color: #f59e0b;
}

/* product-list 的直接子元素 */
.product-list > article {
  min-width: 0;
}

8.3 属性和状态选择器

input[type="email"] {
  border-color: #94a3b8;
}

button:hover {
  background-color: #1d4ed8;
}

input:focus {
  outline: 3px solid #93c5fd;
}

button:disabled {
  cursor: not-allowed;
  opacity: 0.5;
}

不要完全移除键盘焦点样式。用户需要知道当前焦点在哪个元素上。

9 层叠、继承与优先级

当多条规则同时匹配一个元素时，浏览器需要决定最终使用哪条规则，这就是层叠。

p {
  color: gray;
}

.description {
  color: #1f2937;
}

如果一个段落具有 description 类，类选择器比标签选择器更具体，因此最终使用第二种颜色。

可以把优先级简单理解为：

行内样式 > ID 选择器 > 类/属性/状态选择器 > 标签选择器

当优先级相同时，后出现的规则覆盖先出现的规则。

部分属性会从父元素继承，例如 color 和 font-family：

body {
  color: #1f2937;
  font-family: system-ui, sans-serif;
}
提示减少样式冲突

优先使用简短、可复用的类选择器，保持组件样式结构清晰。尽量不要依赖很长的嵌套选择器，也不要频繁使用 !important。

10 盒模型

浏览器把大部分元素看作一个矩形盒子。盒模型从内到外包括：

  1. content：内容区域
  2. padding：内容与边框之间的内边距
  3. border：边框
  4. margin：元素与其他元素之间的外边距
.product-card {
  width: 320px;
  padding: 20px;
  border: 1px solid #e5e7eb;
  margin: 16px;
}

默认情况下，width 只表示内容宽度，内边距和边框会额外增加实际尺寸。通常会统一设置：

*,
*::before,
*::after {
  box-sizing: border-box;
}

使用 border-box 后，声明的宽度会包含内边距和边框，尺寸更容易计算。

10.1 常见尺寸单位

单位 说明
px 固定像素值
% 相对于包含块的比例
rem 相对于根元素字体大小
em 相对于当前元素字体大小
vw、vh 相对于视口宽度、高度

正文和间距常使用 rem，容器宽度常结合 %、max-width 使用：

.page-container {
  width: min(100% - 32px, 1200px);
  margin-inline: auto;
}

11 Flexbox 一维布局

Flexbox 适合处理一行或一列中的排列、间距和对齐。

<div class="toolbar">
  <h2>商品列表</h2>
  <button type="button">新增商品</button>
</div>
.toolbar {
  display: flex;
  align-items: center;
  justify-content: space-between;
  gap: 16px;
}

11.1 主轴与交叉轴

默认 flex-direction: row：

  • 主轴是水平方向，由 justify-content 控制
  • 交叉轴是垂直方向，由 align-items 控制

改为纵向排列：

.product-card {
  display: flex;
  flex-direction: column;
  gap: 12px;
}

11.2 常用属性

.container {
  display: flex;
  flex-wrap: wrap;
  justify-content: center;
  align-items: stretch;
  gap: 24px;
}

.item {
  flex: 1 1 240px;
}
  • flex-wrap：空间不足时是否换行
  • gap：子元素之间的间距
  • flex-grow：是否占用剩余空间
  • flex-shrink：空间不足时是否缩小
  • flex-basis：元素的初始尺寸

12 Grid 二维布局

Grid 适合同时控制行和列，例如响应式商品列表。

<section class="product-grid">
  <article class="product-card">商品一</article>
  <article class="product-card">商品二</article>
  <article class="product-card">商品三</article>
</section>
.product-grid {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  gap: 24px;
}

1fr 表示网格中的一份可用空间。下面的写法可以根据容器宽度自动调整列数：

.product-grid {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(240px, 1fr));
  gap: 24px;
}

每列最小宽度是 240px，空间足够时会自动增加列数。

注记Flexbox 还是 Grid
  • 主要处理一行或一列：优先考虑 Flexbox
  • 同时处理行和列：优先考虑 Grid
  • 两者可以组合使用，不需要二选一

13 定位

position 可以改变元素参与布局的方式。

13.1 相对定位与绝对定位

商品卡片右上角的状态标签可以这样实现：

<article class="product-card">
  <span class="product-badge">新品</span>
  <h2>机械键盘</h2>
</article>
.product-card {
  position: relative;
}

.product-badge {
  position: absolute;
  top: 12px;
  right: 12px;
}

绝对定位元素会相对于最近的、设置了非 static 定位的祖先元素进行定位。

13.2 固定与粘性定位

.back-to-top {
  position: fixed;
  right: 24px;
  bottom: 24px;
}

.page-nav {
  position: sticky;
  top: 0;
}
  • fixed：相对于浏览器视口固定
  • sticky：滚动到指定位置后保持吸附

不要依赖绝对定位完成整个页面布局。常规布局应优先使用正常文档流、Flexbox 或 Grid。

14 响应式设计

响应式页面会根据屏幕或容器尺寸调整布局。

首先确保 HTML 中包含视口配置：

<meta name="viewport" content="width=device-width, initial-scale=1.0" />

14.1 移动优先

先编写小屏幕样式，再使用媒体查询增强大屏幕布局：

.product-grid {
  display: grid;
  grid-template-columns: 1fr;
  gap: 16px;
}

@media (min-width: 640px) {
  .product-grid {
    grid-template-columns: repeat(2, 1fr);
  }
}

@media (min-width: 1024px) {
  .product-grid {
    grid-template-columns: repeat(3, 1fr);
    gap: 24px;
  }
}

14.2 响应式图片

img {
  display: block;
  max-width: 100%;
  height: auto;
}

图片宽度不会超过容器，同时保持原始宽高比。

14.3 不只考虑设备宽度

响应式设计还要考虑：

  • 文本放大后是否被截断
  • 导航是否支持键盘操作
  • 按钮是否有足够大的点击区域
  • 内容是否依赖鼠标悬停才能访问
  • 深色模式下是否保持足够对比度

15 从 HTML/CSS 到 React JSX

React 组件使用 JSX 描述页面结构。JSX 与 HTML 很相似，但有几个重要区别。

15.1 class 改为 className

function ProductCard() {
  return (
    <article className="product-card">
      <h2>机械键盘</h2>
    </article>
  );
}

CSS 文件仍然使用普通类选择器：

.product-card {
  padding: 20px;
}

15.2 标签必须闭合

<img src="keyboard.jpg" alt="机械键盘" />
<input type="text" />

15.3 属性使用驼峰命名

<label htmlFor="product-name">商品名称</label>
<input id="product-name" maxLength={30} />
  • HTML 的 for 在 JSX 中写成 htmlFor
  • maxlength 写成 maxLength
  • 事件 onclick 写成 onClick

15.4 JavaScript 表达式写在花括号中

function ProductCard({ product }) {
  return (
    <article className="product-card">
      <h2>{product.name}</h2>
      <p>¥{product.price}</p>
      <button disabled={!product.inStock}>
        {product.inStock ? "加入购物车" : "暂时缺货"}
      </button>
    </article>
  );
}

结构和样式仍然遵循 HTML/CSS 的规则，只是内容可以由 JavaScript 数据动态生成。
