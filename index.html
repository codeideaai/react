React 学习大纲

从 HTML 与 JavaScript 基础开始

一份面向 React 初学者的 8～10 周学习路线

适合人群：掌握基础 HTML、CSS 和 JavaScript，希望系统学习 React 的初学者。

建议周期：8～10 周，每周学习 5 天，每天 1～2 小时。进度可以按自己的时间调整。

1 一、学习目标

完成本学习大纲后，你应该能够：

  • 理解 React 的组件化思想和基本工作方式
  • 使用 JSX、组件、Props、State 和常用 Hooks 开发页面
  • 正确处理表单、事件、列表渲染和组件通信
  • 使用 React Router 开发多页面体验的单页应用
  • 调用后端 API，并处理加载、成功和失败状态
  • 使用 Vite 创建、调试和构建 React 项目
  • 独立完成一个中小型 React 项目

2 二、基础回顾

React 本质上仍然是 JavaScript。需要熟悉以下内容。

2.1 1. HTML 与 CSS

  • 语义化标签：header、main、section、form、button 等
  • 常用表单元素：input、select、textarea
  • CSS 选择器、盒模型和定位
  • Flexbox 和 Grid 基本布局
  • 响应式设计与媒体查询

2.2 2. JavaScript 必备知识

  • 变量：const 和 let
  • 基本类型、数组和对象
  • 函数、箭头函数和回调函数
  • 条件判断、三元运算符和逻辑运算符
  • 数组常用方法：map、filter、find、some、reduce
  • 对象和数组的解构赋值
  • 展开运算符：...
  • 模块化：import 和 export
  • Promise、async/await 和 fetch
  • DOM 与事件的基础概念

2.3 3. 基础练习

  • 使用数组方法实现商品筛选和价格统计
  • 使用 fetch 获取公开 API 数据并显示到页面
  • 使用原生 JavaScript 制作一个待办事项列表

3 三、React 学习路线

4 第一阶段：认识 React 与开发环境（第 1 周）

4.1 学习内容

  • React 是什么，适合解决什么问题
  • 单页应用（SPA）的基本概念
  • 声明式编程与命令式 DOM 操作的区别
  • 安装 Node.js，认识 npm
  • 使用 Vite 创建 React 项目
  • 理解项目中的主要文件：
    • package.json
    • index.html
    • src/main.jsx
    • src/App.jsx
  • 启动开发服务器和执行生产构建
  • 安装 React DevTools

4.2 阶段练习

  • 创建第一个 React 项目
  • 修改页面标题、文本、图片和样式
  • 尝试把一个静态 HTML 页面迁移到 React 中

4.3 阶段目标

能够独立创建并运行 React 项目，知道浏览器中显示的内容是从哪里渲染出来的。

5 第二阶段：JSX 与组件（第 2 周）

5.1 学习内容

  • JSX 的作用和基本语法
  • JSX 与 HTML 的常见区别：
    • className
    • 标签必须闭合
    • JavaScript 表达式使用 {}
    • 样式对象的写法
  • 函数组件
  • 组件的拆分、导入和导出
  • Props：父组件向子组件传递数据
  • children 的基本使用
  • 条件渲染
  • 列表渲染与 key

5.2 阶段练习

  • 编写可复用的按钮组件
  • 编写用户卡片组件，通过 Props 显示不同用户
  • 使用数组渲染商品列表
  • 根据商品库存显示不同提示

5.3 阶段目标

能够把一个页面拆分成多个组件，并通过 Props 复用组件。

6 第三阶段：事件、State 与表单（第 3 周）

6.1 学习内容

  • React 事件处理
  • useState 的基本使用
  • State 更新与页面重新渲染
  • State 不可直接修改的原则
  • 对象、数组类型 State 的更新方式
  • 受控表单：输入框的值由 State 管理
  • 表单提交和基础校验
  • 状态提升：让多个组件共享状态
  • 单向数据流

6.2 阶段练习

  • 计数器
  • 展开与收起面板
  • 登录表单与错误提示
  • 可新增、完成和删除任务的待办事项列表

6.3 阶段目标

能够让页面响应用户操作，并理解“数据变化驱动页面更新”。

7 第四阶段：副作用与数据请求（第 4 周）

7.1 学习内容

  • 什么是组件的副作用
  • useEffect 的基本使用
  • 依赖数组的含义
  • Effect 清理函数
  • 使用 fetch 和 async/await 请求 API
  • 加载、成功、空数据和错误状态
  • 避免常见的重复请求和无限循环
  • 将重复逻辑提取为自定义 Hook

7.2 阶段练习

  • 获取并展示用户列表
  • 根据关键词搜索数据
  • 给请求添加加载提示和错误提示
  • 编写一个简单的 useFetch 自定义 Hook

7.3 阶段目标

能够从后端获取数据，并完整处理请求过程中的各种页面状态。

8 第五阶段：样式与组件设计（第 5 周）

8.1 学习内容

  • 普通 CSS 文件
  • CSS Modules
  • 动态 className 和内联样式
  • 组件职责与合理拆分
  • 展示组件与业务组件的基本区别
  • 组合优于过度配置
  • 可复用组件的 API 设计
  • 基础可访问性：标签、键盘操作和语义化元素

8.2 阶段练习

  • 制作可复用的 Modal、Button 和 Input 组件
  • 给待办应用添加主题切换
  • 实现桌面端与移动端适配

8.3 阶段目标

能够编写结构清晰、样式可维护、具有一定复用性的组件。

9 第六阶段：路由与完整应用结构（第 6 周）

9.1 学习内容

  • 客户端路由的作用
  • 安装和使用 React Router
  • 路由配置与页面组件
  • Link 与编程式导航
  • 动态路由参数
  • 查询参数
  • 嵌套路由与公共布局
  • 404 页面
  • 项目目录组织方式

9.2 阶段练习

  • 创建首页、列表页和详情页
  • 点击列表项进入详情页
  • 添加导航栏和 404 页面

9.3 阶段目标

能够开发具有多个页面、公共布局和详情路由的 React 单页应用。

10 第七阶段：状态管理与性能基础（第 7 周）

10.1 学习内容

  • 局部状态与全局状态的区别
  • useContext 的使用场景
  • useReducer 的基本使用
  • Context 与 Reducer 的组合
  • 状态应该放在哪里
  • React 组件重新渲染的基本规律
  • useMemo、useCallback 和 memo 的作用
  • 先测量再优化，避免过早优化

10.2 可选扩展

  • 了解 Zustand 或 Redux Toolkit
  • 了解 TanStack Query 解决的数据请求与缓存问题

10.3 阶段练习

  • 使用 Context 实现主题或用户信息共享
  • 使用 useReducer 重构复杂待办事项状态
  • 使用 React DevTools 观察组件渲染

10.4 阶段目标

能够根据状态的使用范围选择合适的位置，并理解基础性能问题。

11 第八阶段：工程化、测试与部署（第 8 周）

11.1 学习内容

  • npm 包和 package.json 脚本
  • ESLint 和代码格式化
  • 环境变量
  • 开发环境与生产环境
  • 错误边界的概念
  • React 组件测试的基本思路
  • 使用 Vitest 与 React Testing Library
  • 构建项目并部署到静态托管平台
  • 浏览器控制台和 Network 面板调试

11.2 阶段练习

  • 为表单或待办组件编写基础测试
  • 修复 ESLint 报告的问题
  • 构建并部署一个 React 项目

11.3 阶段目标

能够使用基本工程工具保证代码质量，并把项目发布给别人访问。

12 四、推荐实战项目

建议按照难度逐步完成，不要只看教程而不动手。

12.1 项目 1：待办事项应用

核心功能：

  • 新增、编辑、删除任务
  • 标记完成状态
  • 按全部、未完成、已完成筛选
  • 使用 localStorage 保存数据

重点知识：组件、Props、State、表单、列表渲染、Effect。

12.2 项目 2：电影或图书搜索应用

核心功能：

  • 输入关键词搜索
  • 展示加载、错误和空结果状态
  • 列表页与详情页
  • 收藏功能
  • 分页或加载更多

重点知识：API 请求、路由、自定义 Hook、状态共享。

12.3 项目 3：个人记账应用

核心功能：

  • 添加收入和支出
  • 按日期、类型筛选
  • 计算余额和分类合计
  • 图表展示
  • 本地存储或连接后端 API

重点知识：复杂状态、表单校验、数据计算、组件设计。

13 五、建议的学习方法

  1. 每学完一个概念，立即写一个最小示例。
  2. 不直接复制教程代码，先自己写，再对照修改。
  3. 遇到报错时，先阅读控制台中的第一条错误信息。
  4. 尝试用自己的话解释 Props、State 和 Effect。
  5. 每周至少完成一个可以运行的小项目。
  6. 定期重构旧代码，检查组件是否过大、状态是否放错位置。
  7. 优先学习 React 官方文档中的核心概念，第三方库按项目需要再学。

14 六、每周学习节奏参考

时间 安排
第 1 天 学习概念，运行官方示例
第 2 天 跟着示例编码，理解每一行作用
第 3 天 不看示例独立重写
第 4 天 完成一个相关小练习
第 5 天 整理笔记、修复问题、重构代码
周末 完成阶段项目或复习薄弱知识

16 八、暂时不用急着学习的内容

初学阶段不必一次掌握所有生态工具。完成前面的核心路线后，再根据实际需要学习：

  • TypeScript
  • Next.js 等 React 框架
  • Redux Toolkit、Zustand 等状态库
  • TanStack Query 等服务端状态工具
  • 动画库和大型组件库
  • 服务端渲染、并发特性和复杂性能优化

学习 React 最重要的是理解组件、Props、State、事件和数据流。先用这些核心能力完成项目，再逐步扩展技术栈。
