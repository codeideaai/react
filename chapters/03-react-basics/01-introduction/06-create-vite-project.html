创建 Vite 项目

检查 Node.js 环境，并使用 Vite 创建、安装和启动 JavaScript 版 React 项目

这一篇会完成第一个可以运行的 React 项目。请亲自在终端执行命令，不要只阅读结果。

检查 Node.js 和 pnpm

终端中运行：

node --version
pnpm --version

两个命令都应该输出版本号。如果尚未安装 pnpm，请根据操作系统选择 pnpm 官方安装方式，安装后重新打开终端并再次检查。

重要检查 Vite 的 Node.js 要求

Vite 对 Node.js 的最低版本要求会随主要版本更新。截至本文编写时，Vite 官方入门文档要求 Node.js 20.19+ 或 22.12+。如果出现版本警告，请以 Vite 官方入门文档 的最新要求为准。

初学阶段建议使用同时满足 Vite 和当前 pnpm 要求的 Node.js LTS 版本。如果电脑上有多个项目，再考虑使用 Node.js 版本管理工具。

创建项目

进入希望保存练习项目的目录：

pnpm create vite react-first-app --template react

命令含义：

  • pnpm create vite：运行 Vite 项目创建工具
  • react-first-app：创建的目录名
  • --template react：选择 JavaScript 版 React 模板

本课程先使用 JavaScript 模板，不需要选择 react-ts。

进入项目

cd react-first-app

检查目录：

pwd
ls

应该能看到 package.json、index.html 和 src。

安装依赖

pnpm install

这一步会：

  • 读取 package.json
  • 根据锁文件解析版本
  • 下载项目依赖
  • 创建 node_modules
注记不必全局安装 Vite

Vite 会作为项目开发依赖安装。通过 pnpm dev 使用项目记录的版本，可以减少不同电脑和项目之间的版本差异。

启动开发服务器

pnpm dev

终端会显示本地地址，例如：

http://localhost:5173/

端口可能因为占用而变化，始终使用终端实际显示的地址。

开发服务器运行时，当前终端会被占用。需要执行其他命令时，可以打开另一个终端标签页。

停止服务器：

Ctrl + C

验证热更新

保持开发服务器运行：

  1. 打开 src/App.jsx
  2. 修改页面标题
  3. 保存文件
  4. 观察浏览器自动更新

如果修改后页面没有变化，检查：

  • 文件是否保存
  • 开发服务器是否仍在运行
  • 浏览器是否打开了正确端口
  • 终端和 Console 是否存在错误

可选：在当前空目录创建

如果当前目录专门用于新项目并且是空的，可以使用 .：

pnpm create vite . --template react

执行前必须确认当前目录没有需要保留的同名文件。初学阶段使用新目录名通常更安全。

Create React App 提示

旧教程可能使用：

create-react-app

Create React App 已被弃用。不要为了跟随旧教程重新使用它。本课程采用 React 官方“从零构建”文档列出的 Vite 方案。

常见错误

找不到 node 或 pnpm

command not found: node
command not found: pnpm

根据缺失的命令检查 Node.js 或 pnpm 是否安装，安装后重新打开终端。

Node.js 版本不满足要求

如果出现 engine 或版本警告，先升级 Node.js，不要忽略后继续排查其他问题。

网络安装失败

确认网络连接和 pnpm 配置。不要从不可信来源下载别人打包的 node_modules。

目录已存在

换一个项目名称，或先确认现有目录中的内容是否可以安全处理。不要直接删除不清楚用途的文件。

完成检查
