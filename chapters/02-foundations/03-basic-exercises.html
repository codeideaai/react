基础练习

用三个小项目巩固 HTML、CSS 与 JavaScript

通过商品统计、异步数据列表和待办事项应用，为学习 React 做好准备

前两节分别回顾了 HTML 与 CSS 和 JavaScript 必备知识。这一节不再增加太多新语法，而是通过三个练习把已有知识连接起来。

三个练习会逐步增加难度：

  1. 使用数组方法处理商品数据
  2. 使用 fetch 加载数据并更新页面
  3. 使用原生 JavaScript 完成待办事项应用

这些练习所涉及的问题，在 React 中仍然会遇到。区别在于 React 会帮助我们更清晰地组织组件、状态和页面更新。

1 学习目标

完成本章后，你应该能够：

  • 把一个需求拆分为数据、计算和页面显示
  • 使用 map、filter、find、some 和 reduce
  • 使用函数减少重复代码
  • 使用 fetch、async/await 和 try/catch
  • 处理加载、成功、空数据和错误状态
  • 使用 DOM API 创建和更新页面元素
  • 监听表单、点击和输入事件
  • 使用数组表示应用状态
  • 在不直接修改原数组的情况下更新数据
  • 说明原生 JavaScript 应用迁移到 React 时哪些部分会发生变化

2 开发环境

每个练习都可以放在一个独立目录中：

exercise-1/
├── index.html
├── styles.css
└── app.js

在 HTML 中引入样式和脚本：

注记使用本地开发服务器

第一个练习可以直接打开 HTML 文件。第二个练习使用 fetch 读取本地 JSON，必须通过本地开发服务器访问，不能直接使用 file:// 打开。

可以使用编辑器提供的本地服务器扩展，或者之后学习 Vite 时使用 Vite 开发服务器。

3.3 提取可复用函数

接下来把固定条件改成函数参数：

const filterByMaxPrice = (products, maxPrice) =>
  products.filter((product) => product.price <= maxPrice);

const findProductById = (products, productId) =>
  products.find((product) => product.id === productId);

const calculateTotalPrice = (products) =>
  products.reduce(
    (total, product) => total + product.price,
    0
  );

调用函数：

console.log(filterByMaxPrice(products, 300));
console.log(findProductById(products, 4));
console.log(calculateTotalPrice(products));

函数的输入和输出越明确，后续越容易测试和复用。React 组件中的很多计算逻辑也可以提取成普通 JavaScript 函数。

3.4 扩展任务

  • 按价格从低到高排序，但不能改变原数组
  • 按 category 对商品进行分组
  • 统计每个分类的商品数量
  • 创建一个函数，同时接收关键词、最高价格和库存条件

排序时可以先使用展开运算符创建副本，再对副本调用 sort。sort 会修改调用它的数组，所以不要直接写 products.sort(...)。

先自己完成其他任务，再展开下面的参考代码。

提示查看扩展任务参考实现 
// 1. 按价格从低到高排序，不修改 products
const sortedProducts = [...products].sort(
  (a, b) => a.price - b.price
);

// 2. 按 category 对商品分组
const productsByCategory = products.reduce(
  (groups, product) => {
    const category = product.category;

    return {
      ...groups,
      [category]: [
        ...(groups[category] ?? []),
        product,
      ],
    };
  },
  {}
);

// 3. 统计每个分类的商品数量
const productCountByCategory = products.reduce(
  (counts, product) => ({
    ...counts,
    [product.category]: (counts[product.category] ?? 0) + 1,
  }),
  {}
);

// 4. 同时根据关键词、最高价格和库存条件筛选
const filterProducts = (
  products,
  { keyword = "", maxPrice = Infinity, inStock = null } = {}
) => {
  const normalizedKeyword = keyword.trim().toLowerCase();

  return products.filter((product) => {
    const matchesKeyword = product.name
      .toLowerCase()
      .includes(normalizedKeyword);
    const matchesPrice = product.price <= maxPrice;
    const matchesStock = inStock === null
      || product.inStock === inStock;

    return matchesKeyword && matchesPrice && matchesStock;
  });
};

console.log(sortedProducts);
console.log(productsByCategory);
console.log(productCountByCategory);

console.log(
  filterProducts(products, {
    keyword: "键盘",
    maxPrice: 400,
    inStock: true,
  })
);

inStock 使用三个值表达不同条件：

  • true：只查有库存商品
  • false：只查无库存商品
  • null：不限制库存状态

?? [] 和 ?? 0 分别为第一次出现的分类提供空数组和初始数量。所有操作都会返回新数组或新对象，不会修改原来的 products。

4.5 拆分 JavaScript 任务

在写代码之前，先把需求拆成几个函数：

loadProducts       请求并返回商品数据
createProductCard  根据一个商品创建 DOM 元素
renderProducts     把商品数组显示到页面
setStatus          更新状态提示

这种拆分方式比把所有代码写在一个函数里更容易理解。

4.6 请求数据

const loadProducts = async () => {
  const response = await fetch("./data/products.json");

  if (!response.ok) {
    throw new Error(`请求失败：${response.status}`);
  }

  return response.json();
};

即使 fetch 得到了 HTTP 响应，状态码也可能是 404 或 500，因此要检查 response.ok。

4.7 安全地创建商品卡片

const createProductCard = (product) => {
  const article = document.createElement("article");
  article.className = "product-card";

  const title = document.createElement("h2");
  title.textContent = product.name;

  const description = document.createElement("p");
  description.textContent = product.description;

  const price = document.createElement("p");
  price.className = "product-price";
  price.textContent = `¥${product.price}`;

  const stock = document.createElement("p");
  stock.className = product.inStock
    ? "stock-available"
    : "stock-unavailable";
  stock.textContent = product.inStock ? "有货" : "缺货";

  article.append(title, description, price, stock);
  return article;
};

使用 textContent 显示外部数据，可以避免把数据当作 HTML 解析。不要在不可信数据上直接使用 innerHTML。

4.8 渲染列表

const productList = document.querySelector("#product-list");
const status = document.querySelector("#status");

const setStatus = (message) => {
  status.textContent = message;
};

const renderProducts = (products) => {
  productList.replaceChildren();

  if (products.length === 0) {
    setStatus("暂时没有商品");
    return;
  }

  const fragment = document.createDocumentFragment();

  products.forEach((product) => {
    fragment.append(createProductCard(product));
  });

  productList.append(fragment);
  setStatus(`共加载 ${products.length} 件商品`);
};

DocumentFragment 可以先在内存中组织元素，再一次加入页面。

4.9 组合加载流程

const reloadButton = document.querySelector("#reload-button");

const showProducts = async () => {
  setStatus("正在加载商品……");
  productList.replaceChildren();
  reloadButton.disabled = true;

  try {
    const products = await loadProducts();
    renderProducts(products);
  } catch (error) {
    console.error(error);
    setStatus("商品加载失败，请稍后重试");
  } finally {
    reloadButton.disabled = false;
  }
};

reloadButton.addEventListener("click", showProducts);
showProducts();

finally 中的代码无论成功还是失败都会执行，适合恢复按钮状态。

const loadTodos = () => {
  try {
    const savedTodos = localStorage.getItem("todos");
    return savedTodos ? JSON.parse(savedTodos) : [];
  } catch (error) {
    console.error("读取任务失败", error);
    return [];
  }
};

let todos = loadTodos();
let currentFilter = "all";

localStorage 只能保存字符串，因此保存时使用 JSON.stringify，读取时使用 JSON.parse。

5.7 保存数据

const saveTodos = () => {
  localStorage.setItem("todos", JSON.stringify(todos));
};

5.8 根据条件筛选任务

const getVisibleTodos = () => {
  if (currentFilter === "active") {
    return todos.filter((todo) => !todo.completed);
  }

  if (currentFilter === "completed") {
    return todos.filter((todo) => todo.completed);
  }

  return todos;
};

5.9 创建任务元素

const createTodoElement = (todo) => {
  const item = document.createElement("li");
  item.className = todo.completed
    ? "todo-item completed"
    : "todo-item";
  item.dataset.todoId = todo.id;

  const checkbox = document.createElement("input");
  checkbox.type = "checkbox";
  checkbox.checked = todo.completed;
  checkbox.setAttribute("aria-label", `完成任务：${todo.title}`);

  const title = document.createElement("span");
  title.className = "todo-title";
  title.textContent = todo.title;

  const deleteButton = document.createElement("button");
  deleteButton.type = "button";
  deleteButton.dataset.action = "delete";
  deleteButton.textContent = "删除";
  deleteButton.setAttribute("aria-label", `删除任务：${todo.title}`);

  item.append(checkbox, title, deleteButton);
  return item;
};

5.10 渲染页面

const render = () => {
  const visibleTodos = getVisibleTodos();
  const activeCount = todos.filter((todo) => !todo.completed).length;

  todoList.replaceChildren(
    ...visibleTodos.map(createTodoElement)
  );

  todoCount.textContent = `剩余 ${activeCount} 项`;
  emptyMessage.hidden = visibleTodos.length !== 0;

  filterButtons.forEach((button) => {
    const isActive = button.dataset.filter === currentFilter;
    button.setAttribute("aria-pressed", String(isActive));
  });
};

render 不负责修改数据，只负责根据当前状态更新页面。保持这个边界可以让代码更容易维护。

5.11 添加任务

todoForm.addEventListener("submit", (event) => {
  event.preventDefault();

  const title = todoInput.value.trim();

  if (!title) {
    return;
  }

  const newTodo = {
    id: crypto.randomUUID(),
    title,
    completed: false,
  };

  todos = [...todos, newTodo];
  saveTodos();
  render();

  todoForm.reset();
  todoInput.focus();
});

event.preventDefault() 阻止表单提交后刷新页面。任务标题使用 trim 去掉首尾空格。

5.12 完成和删除任务

可以在列表上统一监听事件，这种方式叫事件委托：

todoList.addEventListener("change", (event) => {
  if (event.target.type !== "checkbox") {
    return;
  }

  const item = event.target.closest("[data-todo-id]");
  const todoId = item.dataset.todoId;

  todos = todos.map((todo) =>
    todo.id === todoId
      ? { ...todo, completed: event.target.checked }
      : todo
  );

  saveTodos();
  render();
});

todoList.addEventListener("click", (event) => {
  const deleteButton = event.target.closest("[data-action='delete']");

  if (!deleteButton) {
    return;
  }

  const item = deleteButton.closest("[data-todo-id]");
  const todoId = item.dataset.todoId;

  todos = todos.filter((todo) => todo.id !== todoId);
  saveTodos();
  render();
});

5.13 切换筛选条件

filterButtons.forEach((button) => {
  button.addEventListener("click", () => {
    currentFilter = button.dataset.filter;
    render();
  });
});

render();

应用启动时必须调用一次 render，把已有数据显示出来。

5.14 观察状态更新流程

每次用户操作都遵循相同流程：

用户操作 → 更新 todos/currentFilter → 保存数据 → render → 页面更新

React 中的核心流程也非常相似：

用户操作 → 更新 State → React 根据 State 重新渲染组件

React 会帮助我们管理重新渲染，因此不需要手动调用 replaceChildren 或创建 DOM 元素。

6 调试方法

完成练习时，不要只依赖肉眼观察页面。

6.1 检查控制台

打开浏览器开发者工具的 Console 面板：

console.log("当前任务", todos);
console.table(products);
console.error("加载失败", error);

如果出现多条错误，优先处理第一条。后面的错误可能只是第一条错误引起的连锁反应。

6.2 检查网络请求

在 Network 面板中确认：

  • 请求地址是否正确
  • HTTP 状态码是否成功
  • 返回内容是否是预期 JSON
  • 请求是否发出了多次
  • 加载时间是否异常

6.3 设置断点

在 Sources 面板中给事件函数或数据处理代码添加断点，逐行观察变量变化。也可以临时在代码中写：

debugger;

开发者工具打开时，代码会在这里暂停。

7 从这些练习过渡到 React

完成原生 JavaScript 练习后，可以把已有概念与 React 对应起来：

原生 JavaScript React
数据变量 State 或 Props
document.createElement JSX
addEventListener onClick、onChange 等 Props
手动调用 render() State 更新后自动重新渲染
更新 textContent 在 JSX 中读取变量
localStorage 仍然可以使用，通常配合 Effect
fetch 仍然使用，通常配合 Effect 或请求库
拆分普通函数 拆分组件和自定义 Hook

学习 React 之后，JavaScript 数据处理能力仍然重要。React 主要改变的是页面如何根据数据更新，而不是替代 JavaScript。
