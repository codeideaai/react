JavaScript 必备知识

学习 React 前需要掌握的核心语法

通过商品列表示例回顾 React 开发中常用的 JavaScript 知识

React 并没有创造一套新的编程语言。组件、状态更新、列表渲染和数据请求，本质上都在使用 JavaScript。

这篇文章不会完整介绍 JavaScript 的所有知识，而是集中讲解学习 React 时最常用的部分。

1 学习目标

完成本章后，你应该能够：

  • 使用 const 和 let 声明变量
  • 读取和更新数组、对象中的数据
  • 编写普通函数、箭头函数和回调函数
  • 使用条件表达式决定页面显示内容
  • 使用数组方法转换、筛选和统计数据
  • 使用解构与展开运算符编写不可变更新
  • 使用 import 和 export 拆分代码
  • 使用 async/await 和 fetch 请求接口
  • 理解原生 DOM 操作与 React 声明式更新的区别

2 准备示例数据

后面的示例会反复使用下面这组商品数据：

const products = [
  { id: 1, name: "键盘", price: 299, inStock: true },
  { id: 2, name: "鼠标", price: 129, inStock: false },
  { id: 3, name: "显示器", price: 1599, inStock: true },
];

每个商品都是一个对象，所有商品组成一个数组。这种数据结构在 React 项目中非常常见。

3 变量：const 与 let

现代 JavaScript 主要使用 const 和 let 声明变量。

const courseName = "React 入门";
let currentLesson = 1;

currentLesson = 2;
  • const 声明的变量不能重新赋值
  • let 声明的变量可以重新赋值
  • 默认优先使用 const，确实需要重新赋值时才使用 let
注记const 不等于完全不可变

const 只表示变量不能指向另一个值。对象内部的属性仍然可能被修改。

const user = { name: "小明" };

user.name = "小红"; // 可以执行，但 React 中通常不这样更新状态
// user = { name: "小红" }; // 报错：不能重新赋值

在 React 中更新对象或数组状态时，通常要创建一个新值，而不是直接修改旧值。这一点会在展开运算符部分继续讲解。

4 基本类型、数组与对象

4.1 常见基本类型

const title = "React 学习"; // string
const price = 299;          // number
const isLogin = true;       // boolean
const selected = null;      // null
let result;                 // undefined

可以使用 typeof 查看大部分值的类型：

console.log(typeof title);   // "string"
console.log(typeof price);   // "number"
console.log(typeof isLogin); // "boolean"

4.2 数组

数组适合保存一组有顺序的数据。

const skills = ["HTML", "JavaScript", "React"];

console.log(skills[0]);     // "HTML"
console.log(skills.length); // 3

4.3 对象

对象使用键值对描述一个具体事物。

const product = {
  id: 1,
  name: "键盘",
  price: 299,
  inStock: true,
};

console.log(product.name);      // "键盘"
console.log(product["price"]); // 299

当属性名保存在变量中时，可以使用方括号访问：

const field = "price";
console.log(product[field]); // 299

5 函数、箭头函数与回调函数

函数把一段可以重复使用的逻辑封装起来。

5.1 普通函数

function formatPrice(price) {
  return `¥${price.toFixed(2)}`;
}

console.log(formatPrice(299)); // "¥299.00"

price 是参数，return 后面的内容是函数返回值。

5.2 箭头函数

箭头函数是 React 代码中非常常见的写法：

const formatPrice = (price) => {
  return `¥${price.toFixed(2)}`;
};

函数体只有一个返回表达式时，可以进一步简写：

const formatPrice = (price) => `¥${price.toFixed(2)}`;

返回对象时，需要用圆括号包住对象：

const createProduct = (name, price) => ({
  name,
  price,
  inStock: true,
});

5.3 回调函数

把一个函数作为参数传给另一个函数，这个函数就叫回调函数。

const names = products.map((product) => product.name);

这里传给 map 的箭头函数就是回调函数。map 会为每个商品调用一次它。

React 的点击事件也会接收回调函数：

function SaveButton() {
  const handleClick = () => {
    console.log("保存成功");
  };

  return <button onClick={handleClick}>保存</button>;
}

注意传递的是函数 handleClick，不是立即执行后的结果 handleClick()。

6 条件判断与逻辑运算符

6.1 if 判断

function getStockText(product) {
  if (product.inStock) {
    return "有货";
  }

  return "缺货";
}

6.2 三元运算符

三元运算符适合根据一个条件在两个值之间选择：

const stockText = product.inStock ? "有货" : "缺货";

在 JSX 中经常使用它：

<span>{product.inStock ? "有货" : "缺货"}</span>

6.3 逻辑与 &&

只希望条件成立时显示内容，可以使用 &&：

{product.inStock && <button>加入购物车</button>}

6.4 空值合并 ??

当左侧是 null 或 undefined 时，使用右侧默认值：

const displayName = user.nickname ?? "未设置昵称";

它与 || 不完全相同。|| 还会把 0、空字符串和 false 当成需要使用默认值的情况。

7 数组常用方法

React 经常需要把数组转换为组件列表，或者根据条件生成一个新数组。下面这些方法非常重要。

7.1 map：转换每一项

map 会返回一个长度相同的新数组。

const names = products.map((product) => product.name);

console.log(names); // ["键盘", "鼠标", "显示器"]

React 使用 map 渲染列表：

function ProductList() {
  return (
    <ul>
      {products.map((product) => (
        <li key={product.id}>
          {product.name}：¥{product.price}
        </li>
      ))}
    </ul>
  );
}

7.2 filter：筛选数据

filter 保留回调函数返回 true 的元素。

const availableProducts = products.filter(
  (product) => product.inStock
);

删除一项数据时也常使用 filter：

const remainingProducts = products.filter(
  (product) => product.id !== 2
);

7.3 find：找到一项

find 返回第一个符合条件的元素；找不到时返回 undefined。

const product = products.find((product) => product.id === 2);

console.log(product?.name); // "鼠标"

?. 是可选链。左侧为 null 或 undefined 时，它会返回 undefined，避免继续访问属性导致报错。

7.4 some：判断是否存在

只需要知道是否至少有一项符合条件时，使用 some：

const hasExpensiveProduct = products.some(
  (product) => product.price > 1000
);

console.log(hasExpensiveProduct); // true

7.5 reduce：累计计算

reduce 可以把一个数组汇总为一个值：

const totalPrice = products.reduce(
  (total, product) => total + product.price,
  0
);

console.log(totalPrice); // 2027

这里的 0 是累计值 total 的初始值。第一次执行回调时，total 为 0。

提示如何选择数组方法
  • 每一项都要转换：使用 map
  • 筛选出多项：使用 filter
  • 查找某一项：使用 find
  • 判断是否存在：使用 some
  • 汇总为一个结果：使用 reduce

8 解构赋值

解构可以从对象或数组中快速取出需要的值。

8.1 对象解构

const product = {
  id: 1,
  name: "键盘",
  price: 299,
};

const { name, price } = product;

console.log(name);  // "键盘"
console.log(price); // 299

React 组件经常直接解构 Props：

function ProductCard({ name, price, inStock }) {
  return (
    <article>
      <h2>{name}</h2>
      <p>¥{price}</p>
      <p>{inStock ? "有货" : "缺货"}</p>
    </article>
  );
}

可以设置默认值，也可以给属性换一个变量名：

const { name: productName, discount = 0 } = product;

8.2 数组解构

const colors = ["红色", "绿色", "蓝色"];
const [firstColor, secondColor] = colors;

React 的 useState 就使用了数组解构：

const [count, setCount] = useState(0);

9 展开运算符

展开运算符 ... 可以把数组元素或对象属性展开到一个新值中。

9.1 复制并更新数组

const newProduct = {
  id: 4,
  name: "耳机",
  price: 399,
  inStock: true,
};

const nextProducts = [...products, newProduct];

9.2 复制并更新对象

const product = products[0];

const updatedProduct = {
  ...product,
  price: 259,
};

后面的 price 会覆盖前面展开出来的旧 price。

9.3 更新数组中的某个对象

const nextProducts = products.map((product) =>
  product.id === 1
    ? { ...product, price: 259 }
    : product
);

这种写法没有修改原数组和原对象，而是生成了新数组，并为需要变化的商品创建新对象。这是更新 React State 的常用方式。

重要避免直接修改 React State

不要使用 products.push(...) 或直接写 product.price = 259 来更新 State。React 通常依靠新旧值的引用变化判断是否需要重新渲染。

10 模块化：export 与 import

模块化可以把代码拆分到不同文件中。

10.1 命名导出

// utils.js
export const formatPrice = (price) => `¥${price.toFixed(2)}`;

export const isExpensive = (price) => price > 1000;

导入时使用相同名称和花括号：

// ProductCard.jsx
import { formatPrice, isExpensive } from "./utils.js";

10.2 默认导出

// ProductCard.jsx
export default function ProductCard() {
  return <article>商品卡片</article>;
}

默认导入不使用花括号，名称可以自行决定：

import ProductCard from "./ProductCard.jsx";

一个模块只能有一个默认导出，但可以有多个命名导出。

11 Promise、async/await 与 fetch

浏览器请求接口需要时间，JavaScript 不会停下来一直等待，而是使用 Promise 表示“将来才会得到的结果”。

11.1 使用 async/await

async function loadProducts() {
  const response = await fetch("https://example.com/api/products");

  if (!response.ok) {
    throw new Error(`请求失败：${response.status}`);
  }

  const data = await response.json();
  return data;
}
  • async 表示函数会返回 Promise
  • await 等待 Promise 完成并取得结果
  • response.ok 用来判断 HTTP 状态是否成功
  • response.json() 也是异步操作，因此也需要 await

11.2 处理错误

async function showProducts() {
  try {
    const products = await loadProducts();
    console.log(products);
  } catch (error) {
    console.error("加载商品失败", error);
  }
}

在 React 中，通常要分别保存加载、成功和失败状态：

const [products, setProducts] = useState([]);
const [isLoading, setIsLoading] = useState(true);
const [error, setError] = useState(null);

具体如何在组件中发起请求，会在学习 useEffect 时继续讲解。

12 DOM、事件与 React 的区别

使用原生 JavaScript 时，我们会主动查找并修改 DOM：

const button = document.querySelector("#counter-button");
const result = document.querySelector("#result");
let count = 0;

button.addEventListener("click", () => {
  count += 1;
  result.textContent = count;
});

React 更关注“当前数据应该对应什么界面”：

function Counter() {
  const [count, setCount] = useState(0);

  return (
    <div>
      <p>{count}</p>
      <button onClick={() => setCount(count + 1)}>增加</button>
    </div>
  );
}

点击按钮后，我们更新 count，React 根据新的 count 重新生成界面。通常不需要自己调用 querySelector 或修改 textContent。

事件名称也有一些区别：

原生 HTML / DOM React JSX
onclick onClick
onchange onChange
class className
传入字符串代码 传入 JavaScript 函数
